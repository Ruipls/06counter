# 计数助手 UX/UI 体验优化方案

**分支**：`v2-ui-polish`（自 `v2-sub-mode` 切出）
**原则**：不改变产品逻辑 —— 激活体系、localStorage 数据结构、计数/库存逻辑、单文件架构、防缩放体系、事件模型全部保持不变。只动呈现层（CSS / 文案 / DOM 结构 / 少量纯 UI 的 JS）。

**设计目标回归场景**：摊主在户外（强光/夜市）、单手、忙碌中操作。一切优化围绕三个词：**看得清、点得准、错得起**。

---

## 一、问题清单与优化点

### A. 计数页（核心页面，优先级最高）

#### A1. 长按撤销不可发现 · P0
**现状**：长按 500ms 是 -1（加法模式）/ 退货 +1（减法模式）的唯一入口（`onBtnPointerDown`, L1273），界面上零提示。误点了 +1 的新用户无法自救。
**方案**：
1. 长按期间给视觉反馈：按住 ~150ms 后按钮出现环形进度或自下而上的深色填充，到 500ms 触发时"充满"。用户从此能感知"按住有事发生"。纯 CSS 动画 + 现有 timer，不改判定逻辑。
2. 首次使用提示：首次进入计数页显示一次性提示条"点一下 +1，按住 0.5 秒 −1"，点击或 5 秒后消失，`localStorage` 记一个 `hintSeen` 标记。

#### A2. 误触后缺少快捷撤销 · P0
**现状**：连续快速点击时点错产品，需要找到那个按钮再长按 500ms。
**方案**：每次 +1/−1 后，toast 改为显示 `产品一 +1 [撤销]`，2.5 秒内可点撤销。撤销 = 对同一产品执行一次反向操作（复用现有逻辑，`todayClicks` 照常记录一条反向流水，数据模型不变）。

#### A3. 自定义颜色下文字对比度不足 · P0
**现状**：按钮文字固定白色（`.product-btn { color:#fff }`, L105），用户可用原生取色器选任意浅色（如黄色、浅粉），强光下完全看不清。
**方案**：`renderGrid()` 时按背景色亮度（YIQ 公式）自动切换黑/白文字与对应 text-shadow。约 10 行 JS。

#### A4. 点击闪烁反馈在浅色按钮上失效 · P0
**现状**：+1 反馈是 `brightness(1.8)`（L151），浅色底几乎无变化；−1 的 `brightness(0.3)` 尚可。
**方案**：改用伪元素覆盖层闪烁（+1 白色半透明层，−1 黑色半透明层）+ 现有缩放弹跳，任何底色都可感知。只改 CSS keyframes。

#### A5. 按钮信息层级重排 · P1
**现状**：数字 1.8em，名称 0.9em，售出额 0.65em，价签 0.55em 挤在右上角。
**方案**：数字加大到 ~2.2em（摊主 1 米外扫一眼要能读）；名称保持；"¥金额"与价签合并到底部一行弱化显示，右上角腾空，减少视觉噪音。

#### A6. 空槽位语义不明 · P1
**现状**：空槽只有一个"+"（L1238），点击跳到设置页顶部（版本信息区），用户茫然。
**方案**：文案改为"＋ 添加产品"；点击后跳设置页并**滚动到产品管理区、聚焦名称输入框**。仍是 `switchTab('settings')`，只加一个 scroll + focus。

#### A7. 减法模式未设库存的引导断层 · P1
**现状**：未设库存按钮显示"--"，点击只弹 toast"请先设置初始库存"，而库存设置入口在设置页里（`btnStockSetupSettings`），链路断了。
**方案**：a) 点击"--"按钮直接弹出现有的库存设置弹窗（`openStockSetup()`，入口前移，逻辑不变）；b) 减法模式且全部未设库存时，网格顶部显示引导卡"📦 今天带了多少货？先登记库存"。

### B. 顶部栏

#### B1. 信息拥挤 + 收入隐私 · P1
**现状**：版本徽标 + 模式徽标 + 日期挤在 13px 一行；收入金额常驻显示——顾客付款时凑近看手机，摊主未必想让人看到当日总收入。
**方案**：日期与徽标合并弱化为一行小字；**点击收入数字可切换为"¥ ∗∗∗"打码显示**，状态存 settings。这是摊主场景的真实需求，成本一个 click handler。

#### B2. 减法模式下标签写死 · P2
**现状**：HTML 里 `topbar-stat-label` 固定"收入"，减法模式下值变成"售出 ¥…"，标签语义重复（L680 + L1173）。
**方案**：label 随模式动态切换：加法「收入」/ 减法「已售」，值只放数字。

### C. 设置页

#### C1. 区块顺序倒置 · P0
**现状**：版本信息（激活推销）排第一屏，最高频的"产品管理"沉底（L694-751）。与 PRD"免费版必须好用"的原则冲突。
**方案**：重排为 **产品管理 → 计数模式 → 布局调整 → 外观 → 版本信息**。升级弹窗"输入激活码"跳转本来就带滚动定位，不受影响。

#### C2. 计数模式文案抽象 · P0
**现状**："➕ 加法 / ➖ 减法"是开发视角的术语；且切换影响整个计数页语义却无任何说明。
**方案**：文案改为「**卖出计数** — 卖一件点一下，从 0 累加」/「**库存倒数** — 先登记库存，卖一件减一件」；切换后 toast 一句话说明当前模式行为。纯文案。

#### C3. 布局编辑模式缺少"在场"反馈 · P1
**现状**：设置页打开开关 → 切回计数页 → 只有右下角小 handle 提示编辑态，没有出口说明，也不知道长按可换位。
**方案**：编辑模式下计数页顶部显示常驻横幅：「布局编辑中 · 拖角调大小 / 长按拖动换位 **[完成]**」，点完成即关闭开关回到计数。开关状态与现有 `layoutEditMode` 同源。

#### C4. 颜色选择体验差 · P1
**现状**：仅原生 `<input type=color>`（L747），移动端取色器难用，且易选出低对比度颜色（连锁引发 A3）。
**方案**：添加/编辑表单提供 10 个精选预设色块（与白字对比度达标的饱和色系），一点即选；保留原生取色器作"自定义"入口。

#### C5. 模板选择器图标无意义 · P2
**现状**：⊞ / ⊟ / ⊟⊟ 字符（L704-707）无法传达 6/10/20/40 的差异。
**方案**：改为迷你网格缩略图（内联 SVG 或 CSS grid 画 2×3、2×5、4×5、4×10 的小方块示意）。仓库中未使用的 `icon-grid.svg` 可作素材参考。

#### C6. 编辑/删除按钮触控目标偏小 · P2
**现状**：`.btn-sm` padding 4px 10px，实际高度 <30px，低于 44×44pt 最小触控标准。
**方案**：产品条目的操作按钮增大到 ≥36px 高，条目本身整行可点进入编辑，删除仅保留图标按钮 + 确认。

### D. 记录页

#### D1. "重置今日"危险操作的确认体验 · P0
**现状**：已有原生 `confirm()`（L2020），但系统弹窗样式与 App 割裂，且按钮与"导出 CSV"并排等宽（L758-761），视觉权重相同。
**方案**：换成 App 统一样式的确认弹窗（复用现有 modal 组件），文案带当日数量「将清空今日 N 件记录，不可恢复」；按钮降权为红色文字次要样式，与导出拉开层级。

#### D2. 当日数据可见性 · P1
**现状**：记录页只有归档历史，"今天卖了什么"要回计数页逐格看。
**方案**：记录页顶部渲染"今日（实时）"卡片，复用 `history-day` 卡片样式渲染 `todayCounts/todayClicks`，纯展示不动数据。

### E. 底部导航与全局

#### E1. 导航图标抽象 · P1
**现状**：● ◎ ○ 三个圆（L766-774）无语义，靠文字辨认。
**方案**：换内联 SVG：计数=四宫格、设置=齿轮、记录=时钟/列表。保持 20px 尺寸与现有 HIG 适配不变。

#### E2. 设计 token 收敛 · P1
**现状**：CSS 变量基础好，但间距、字号、阴影仍有散落的魔法数（4px 阴影两套、字号 9–20px 十余档）。
**方案**：固化刻度——间距 4/8/12/16/20，字号档位 11/13/15/17 + count 特大号，阴影两级（卡片/浮层）。为后续所有 UI 迭代打底。

#### E3. 深色模式 · P2
**场景**：夜市摊主晚间使用，白底刺眼。
**方案**：`prefers-color-scheme: dark` + 现有 CSS 变量即可低成本实现（约 15 个变量值），另在外观设置加手动开关（跟随系统/浅色/深色）。产品按钮本身有色不受影响，改的是壳。

## 二、优先级与实施批次

| 批次 | 内容 | 预估改动 |
|------|------|----------|
| **P0（先做，一次提交一项）** | A1 长按进度 + 首次提示 · A2 toast 撤销 · A3 文字对比度自适应 · A4 闪烁反馈重做 · C1 设置页重排 · C2 模式文案 · D1 重置确认 | 各 10–60 行，互相独立 |
| **P1** | A5 按钮层级 · A6 空槽引导 · A7 库存引导 · B1 收入打码 · C3 编辑模式横幅 · C4 预设色板 · D2 今日卡片 · E1 导航图标 · E2 token 收敛 | 中等 |
| **P2** | B2 动态标签 · C5 模板缩略图 · C6 触控目标 · E3 深色模式 | 择机 |

## 三、明确不改动的部分

激活码/机器码体系、`localStorage` 键与数据结构、`todayClicks` 流水格式、CSV 导出格式、长按 500ms 判定阈值与 pointer 事件模型、iOS 防缩放六层防护（尤其不得拦截 `touchstart`）、PWA/Service Worker、单文件架构。

## 四、验收标准

1. 任意自选颜色的按钮，强光模拟（屏幕最低亮度）下名称与数字可读。
2. 新用户不经指导，30 秒内能发现"长按可减"。
3. 误点 +1 后 3 秒内可一步撤销。
4. "重置今日"无法一次点击完成。
5. 免费版用户打开设置页，第一屏是产品管理而非激活推销。
6. 所有改动在 iOS Safari PWA 模式回归：音效正常（AudioContext 未被破坏）、双击/双指缩放仍被拦截、safe-area 无错位。
