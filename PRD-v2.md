## 一、产品概述

**产品名称**：计数助手

**目标用户**：市集摆摊摊主，需要在售卖过程中快速记录不同产品的销售数量

**核心场景**：
- 摊主在忙碌的售卖中，单手快速点击产品按钮 +1 计数
- 收摊后查看当天收入、导出 CSV 记账
- 查看历史销售记录

**分发策略**：
- 免费版通过微信群/朋友圈传播，功能完整但产品数量受限
- 专业版通过激活码解锁，微信社群直销
- 后续上架 App Store 扩展全球市场

### 设计原则

- **免费版必须好用**：限制仅在产品数量，其余功能完整。用户用着舒服才会传播。
- **升级路径自然**：生意越好、产品越多，越需要专业版。限制点在用户成长路径上。
- **不拆分文件**：同一个 `index.html`，通过激活状态切换行为。方便维护和传播。
- **一人一码**：激活码绑定设备，防止私下传递和共用。

## 三、激活码系统（机器码绑定）

### 整体流程

```
用户设备                          客服（你）
─────────                        ─────────
打开 App 设置页
看到 8 位设备码
   │                              ← 微信发送设备码 →
   │                             打开 generator.html
   │                             输入设备码 → 生成激活码
   │                              ← 微信发回激活码 →
输入激活码
验证通过 → 解锁专业版
```

### 设备码（机器码）

- 由设备指纹生成：CPU 核心数、平台、语言、屏幕色深、触摸点数、时区偏移
- 8 位大写 hex 字符，适合微信发送
- 同一设备每次生成一致，不同设备不同

### 激活码

- 格式：`CP-XXXX-XXXX-XXXX`（12 位 hex，大写）
- 生成算法：`simpleHash(机器码 + 密钥)` 取前 12 位
- 验证方式：客户端用相同算法重算，比对激活码
- 绑定设备：只在生成时对应的设备上有效

### 密钥

- 内置于 `index.html` 和客服工具 `generator.html` 中
- 变量名 `PRO_KEY`，值为随机字符串
- 不提交到公开仓库（`.claude/` 已 gitignore）

### 客服工具

- 文件：`.claude/tools/generator.html`（本地使用，不提交 git）
- 功能：输入 8 位设备码 → 一键生成激活码 → 复制发送

### 安全性说明

- 前端验证无法做到绝对安全，但目标用户（摊主）无逆向动机
- 机器码绑定从根本上解决激活码分享问题：分享给别人的设备无法激活
- 激活码 = 设备码的签名，无法从激活码反推设备码

## 四、功能需求详述

### 4.1 激活入口

**位置**：设置页 → "版本信息"区块（外观设置下方）

**免费版界面**：
- 当前版本标识（灰色"免费版"）
- 功能限制说明（5 产品 / 5 宫格）
- **设备码展示**（8 位，引导用户发给客服）
- 激活码输入框（placeholder: `CP-XXXX-XXXX-XXXX`）
- "激活"按钮
- 引导文案："购买后将上方设备码发给客服，获取专属激活码"

**专业版界面**：
- "专业版" + 金色 PRO 徽标
- 激活码信息（脱敏显示，如 `CP-1348-***-***C53`）
- 已解锁功能说明

**交互**：
- 输入激活码，提交时自动转大写比对
- 验证成功 → toast "已解锁专业版！" → 刷新 UI
- 验证失败 → toast "激活码无效"

### 4.2 产品数量限制

**限制规则**：免费版最多添加 5 个产品

**触发场景**：

| 场景 | 行为 |
|------|------|
| 添加第 6 个产品 | 弹出升级提示弹窗，阻止添加 |
| 切换到 10/15/20 宫格模板 | 弹出升级提示弹窗，阻止切换 |

**升级提示弹窗**：
- 标题："专业版功能"
- 正文：说明免费版限制（最多 5 个产品 / 仅 5 宫格），引导购买激活码
- 按钮：「输入激活码」（跳转到设置页版本区块）、「暂不升级」（关闭弹窗）
- 点击「输入激活码」→ 切到设置页 → 滚动到版本信息区块

### 4.3 顶部栏状态

- 日期左侧显示版本标记
- 免费版：灰色徽标 "免费版"
- 专业版：金色徽标 "PRO"

### 4.4 模板选择器

- 免费版：5 宫格正常可选，10/15/20 宫格显示半透明 + 锁定图标
- 专业版：全部可选
- 点击锁定项 → 弹出升级提示

### 4.5 激活状态管理

- `localStorage` 中存储三个字段：
  - `counter-pro-activated`: `"true"` / 不存在
  - `counter-pro-code`: 完整激活码（用于脱敏显示）
  - `counter-pro-machine`: 设备码（用于验证）
- 清除浏览器数据会导致激活状态丢失
- 由于机器码稳定生成，重新输入相同激活码仍然有效
- 初始化时从 localStorage 读取，无需网络请求

### 4.6 免费版模板强制

- `loadState()` 中检查：如果不是专业版且已保存模板 > 5，强制重置为 5 宫格
- 防止专业版降级后残留大模板数据

### 4.7 点击反馈系统（防误触）

**设计目标**：每次点击操作必须有即时、可感知的反馈，防止在忙碌售卖场景中的误触和漏触。

**反馈层级**：

| 层级 | 机制 | +1（短按） | -1（长按 500ms） | iOS 静音 | Android 静音 |
|------|------|:---:|:---:|:---:|:---:|
| 视觉 | CSS 关键帧动画 | 绿色高亮闪烁 + 缩放弹跳 | 深暗闪烁 + 缩放弹跳 | 正常 | 正常 |
| 音效 | Web Audio API | 1200Hz 正弦波 "叮"(80ms) | 400Hz 三角波 "咚"(120ms) | 被静音开关屏蔽 | 正常 |
| 震动 | navigator.vibrate | 10ms 短震 | [10,30,10] 模式长震 | 不支持 | 正常 |

**视觉动画细节**：
- +1（flashGreen）：亮度峰值 1.8x → 弹回 1.15x → 归位，缩放 0.88 → 1.03 → 1.0
- -1（flashRed）：亮度谷值 0.3x → 弹回 0.85x → 归位，缩放同上
- 动画时长 300ms，ease 缓动

**音效开关**：
- 设置页 → 外观设置 → "点击音效"开关
- 默认开启，状态持久化至 `localStorage`（`settings.soundEnabled`）
- 关闭时音效静默，视觉和震动不受影响

**AudioContext 健壮性**：
- `visibilitychange` 监听：页面恢复可见时自动 resume
- `pageshow` 监听：bfcache 恢复时重建 AudioContext（`e.persisted === true`）
- `getAudioCtx()` 每次调用检测 `closed`/`suspended` 状态，按需重建
- 注意：`touchstart` 的 `preventDefault()` 会破坏 iOS 用户手势标志 → AudioContext 无法启动

### 4.8 iOS Safari 防缩放体系

**问题**：iOS Safari 的双击缩放、双指缩放、双击长按缩放会干扰计数的点击操作。

**6 层防护**：

| 层 | 实现 | 拦截目标 |
|---|------|---------|
| viewport meta | `user-scalable=no` + `maximum-scale=1.0` | 声明禁止缩放 |
| CSS body | `touch-action: manipulation` | 双击缩放 |
| CSS .grid-container | `touch-action: manipulation` | 容器级双击缩放（DOM 重建后仍生效） |
| CSS .product-btn | `touch-action: manipulation` | 按钮级双击缩放 |
| JS touchend | 500ms 窗口内第二次 touchend → preventDefault | 快速双击 |
| JS dblclick | preventDefault | iOS 合成双击事件 |
| JS gesturestart | preventDefault | 双指缩放 |

**关键约束**：不能拦截 `touchstart`——iOS Safari 会将 `touchstart` 的 `preventDefault()` 标记为"非用户手势"，导致 `AudioContext.resume()` 静默失败。

## 五、技术方案

### 技术栈

- 纯 HTML/CSS/JavaScript，单文件，无框架依赖
- 哈希算法：纯 JS `simpleHash()`（双哈希组合，兼容所有协议）
- 存储：localStorage
- 部署：GitHub Pages（免费静态托管）

### 哈希函数

```
simpleHash(s):
  hash1 = 逐字符移位累加 (类似 Java String.hashCode)
  hash2 = 逐字符 FNV-1a
  返回: hash1(8hex) + hash2(8hex) = 16位 hex
```

- 纯 JS 实现，不依赖 Web Crypto API
- `file://`、`http://`、`https://` 均可运行

### 代码结构

```
index.html
├── CSS: 版本徽标、升级弹窗、锁定模板、版本信息区块
├── HTML: 版本信息区块 (#versionInfo)、升级弹窗 (#modalUpgrade)、顶部栏徽标
└── JS:
    ├── PRO_KEY — 密钥常量（勿提交公开仓库）
    ├── simpleHash(s) — 双哈希函数
    ├── generateMachineCode() — 设备指纹 → 8 位设备码
    ├── makeActivationCode(machineCode) — 机器码 → CP-XXXX-XXXX-XXXX
    ├── isPro() — 读取 localStorage 判断是否专业版
    ├── activatePro(code) — 验证激活码并写入状态
    ├── renderVersionInfo() — 渲染版本信息（含设备码/激活入口）
    ├── showUpgradeModal() — 显示升级提示弹窗
    ├── getAudioCtx() — Web Audio API 上下文管理（锁屏/bfcache 自动恢复）
    ├── playIncSound() — +1 音效（1200Hz 正弦波，80ms）
    ├── playDecSound() — -1 音效（400Hz 三角波，120ms）
    ├── visibilitychange / pageshow 监听 — iOS 唤醒后 AudioContext 恢复
    ├── touchend / dblclick / gesturestart 拦截 — iOS Safari 防缩放体系
    └── 修改 btnAdd / templateGrid 逻辑加入限制检查
```

## 六、风险点与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 激活码被分享 | 无影响 | 机器码绑定，其他设备无法使用 |
| 用户更换手机 | 设备码变化，需重新激活 | 客服免费补发（体现服务） |
| 清除浏览器数据 | 激活状态丢失 | 重新输入相同激活码即可恢复 |
| localStorage 被篡改 | 可手动设 pro=true | 前端无法防御，目标用户无此动机 |
| 设备指纹碰撞 | 两设备生成相同机器码 | 极低概率，8 位 hex 空间足够 |
